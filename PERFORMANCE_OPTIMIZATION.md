# 系统性能优化方案

## 1. 核心问题分析

### 1.1 VideoCard 组件过于复杂

- **问题**：VideoCard 组件包含过多功能和状态，导致渲染和更新开销大
- **表现**：
  - 包含多个子组件（MobileActionSheet、VideoDetailPreview、CombinedDetailModal）
  - 多个 useEffect 钩子监听不同状态变化
  - 复杂的条件渲染和样式逻辑
  - 频繁的状态更新和重渲染

### 1.2 虚拟滚动优化不足

- **问题**：VirtualSearchGrid 组件的预加载策略可能导致内存占用过高
- **表现**：
  - 预加载 30 张图片，可能导致大量网络请求和内存占用
  - 加载更多的批处理大小（LOAD_MORE_BATCH_SIZE=8）可能过大

### 1.3 网络请求优化不足

- **问题**：搜索和详情加载时可能存在过多并行请求
- **表现**：
  - 搜索结果页同时加载大量数据
  - 视频详情页并行请求豆瓣数据和搜索数据
  - 缺少有效的缓存机制

### 1.4 CSS 动画和过渡效果

- **问题**：大量复杂的 CSS 动画和过渡效果可能导致 GPU 使用率过高
- **表现**：
  - 卡片悬停效果包含多个同时触发的过渡
  - 渐变光泽动画持续运行
  - 大量元素使用 transition-all

## 2. 优化计划与优先级

### 优先级 1：VideoCard 组件优化

#### 2.1.1 组件拆分

- 将 VideoCard 拆分为更小的、职责单一的组件
- 实现组件懒加载

#### 2.1.2 状态管理优化

- 减少组件内部状态数量
- 优化 useEffect 依赖项
- 减少不必要的重渲染

### 优先级 2：虚拟滚动优化

#### 2.2.1 调整预加载策略

- 减少预加载图片数量
- 优化加载更多批处理大小

#### 2.2.2 优化虚拟滚动参数

- 调整 overscanCount
- 优化行高计算

### 优先级 3：网络请求优化

#### 2.3.1 实现缓存机制

- 为搜索结果添加本地缓存
- 为豆瓣详情添加缓存

#### 2.3.2 优化并行请求

- 限制并行请求数量
- 实现请求优先级

### 优先级 4：CSS 动画优化

#### 2.4.1 简化动画效果

- 减少同时运行的动画数量
- 简化渐变光泽动画

#### 2.4.2 使用 CSS 硬件加速

- 为动画元素添加硬件加速
- 优化阴影效果

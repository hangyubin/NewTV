# 短剧数据获取流程分析

## 1. 路由结构

短剧数据通过 `/api/short-drama` 路由获取，该路由使用 Next.js App Router 实现，配置如下：

- 运行时：`nodejs`
- 动态渲染：`force-dynamic`
- 缓存策略：`revalidate: 0`（不缓存）

## 2. 请求处理流程

### 2.1 请求参数解析

API 接收以下查询参数：

| 参数名 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|
| `type` | 字符串 | `all` | 短剧类型筛选（爱情、家庭、现代等） |
| `region` | 字符串 | `all` | 地区筛选（中国大陆、韩国、日本等） |
| `year` | 字符串 | `all` | 年份筛选 |
| `page` | 数字 | `1` | 页码 |
| `limit` | 数字 | `25` | 每页数量 |

### 2.2 配置获取

- 获取系统配置：`const config = await getConfig()`
- 获取可用 API 站点：`const apiSites = await getAvailableApiSites()`

## 3. 搜索策略

### 3.1 关键词生成

使用多个短剧相关关键词进行搜索，包括：

```javascript
const shortDramaKeywords = [
  '短剧', '微剧', '竖屏短剧', '网络短剧', '小剧场',
  '微电影', 'mini drama', 'micro drama', 'short drama',
  'short film', '竖屏', '短视频剧', '网络剧', '迷你剧',
  '短剧集', '短剧精选', '热门短剧', '短剧推荐', '短剧剧场',
  '短剧专区', '微短剧', '短剧热播', '短剧合集', '短剧在线',
  '短剧免费', '短剧大全'
];
```

### 3.2 站点选择

使用所有可用的 API 站点进行搜索，确保获取更多数据。

### 3.3 并行搜索

为每个站点和关键词组合创建搜索请求，并行执行：

```javascript
for (const site of allSites) {
  for (const keyword of allKeywords) {
    searchPromises.push(searchFromApi(site, keyword));
  }
}
```

### 3.4 超时处理

为每个搜索请求设置超时时间（10秒），避免单个站点响应慢影响整体性能：

```javascript
const results = (await Promise.race([
  searchFromApi(site, keyword),
  new Promise((_, reject) =>
    setTimeout(() => reject(new Error(`${site.name} timeout`)), 10000)
  ),
])) as SearchResult[];
```

## 4. 数据过滤

### 4.1 短剧检测

使用 `isShortDrama` 函数检测是否为短剧内容，检测逻辑包括：

- 检查类型名称（type_name）是否包含短剧相关标识
- 检查标题是否包含短剧关键词
- 检查标题长度（短剧通常较短）
- 检查是否包含"短"或"微"关键字

### 4.2 黄色内容过滤

根据系统配置，过滤黄色内容：

```javascript
if (!config.SiteConfig.DisableYellowFilter) {
  const content = `${typeName} ${title} ${desc}`;
  if (yellowWords.some((word: string) => content.includes(word))) {
    return false;
  }
}
```

### 4.3 类型、地区、年份筛选

根据请求参数进行筛选：

- 类型筛选：使用 `getShortDramaType` 函数获取短剧类型
- 地区筛选：使用 `getContentRegion` 函数获取地区信息
- 年份筛选：使用 `matchYear` 函数匹配年份

## 5. 数据处理

### 5.1 去重

使用 `Set` 数据结构根据标题去重：

```javascript
const seenTitles = new Set<string>();
const uniqueResults: SearchResult[] = [];

for (const result of allResults) {
  if (!seenTitles.has(result.title)) {
    seenTitles.add(result.title);
    uniqueResults.push(result);
  }
}
```

### 5.2 排序

按以下规则排序：

1. 优先按年份排序（新的在前）
2. 然后按标题长度排序（短剧通常标题较短）

```javascript
const sortedResults = uniqueResults.sort((a, b) => {
  const yearA = parseInt(a.year) || 0;
  const yearB = parseInt(b.year) || 0;
  if (yearA !== yearB) {
    return yearB - yearA;
  }
  return a.title.length - b.title.length;
});
```

### 5.3 分页

根据请求参数进行分页：

```javascript
const startIndex = (page - 1) * limit;
const endIndex = startIndex + limit;
const paginatedResults = sortedResults.slice(startIndex, endIndex);
```

## 6. 响应返回

返回格式化的 JSON 响应，包含：

- `results`：分页后的短剧数据
- `total`：总数据量
- `page`：当前页码
- `limit`：每页数量
- `totalPages`：总页数

响应头配置：

```javascript
headers: {
  'Cache-Control': `public, max-age=${cacheTime}, s-maxage=${cacheTime}`,
  'CDN-Cache-Control': `public, s-maxage=${cacheTime}`,
  'Vercel-CDN-Cache-Control': `public, s-maxage=${cacheTime}`,
  'Netlify-Vary': 'query',
}
```

## 7. 错误处理

- 捕获搜索过程中的错误，返回空数组
- 处理请求超时，返回空数组
- 捕获整体流程中的错误，返回 500 错误响应

## 8. 优化策略

### 8.1 多级搜索

- 第一级：使用短剧相关关键词搜索
- 第二级：如果没有结果，使用更通用的关键词（热门、最新、精选、推荐）再次搜索

### 8.2 并行请求

- 使用 `Promise.allSettled` 并行处理多个搜索请求
- 限制单个请求超时时间，避免整体流程卡顿

### 8.3 高效去重

- 使用 `Set` 数据结构进行高效去重
- 根据标题去重，确保结果唯一性

## 9. 流程图

```
┌───────────────────────────────────────────────────────────────────┐
│                        短剧数据获取流程                          │
└───────────────────────────────────────────────────────────────────┘

1. 接收请求 → 2. 解析参数 → 3. 获取配置 → 4. 生成搜索关键词
        ↓
5. 遍历站点和关键词 → 6. 并行发送搜索请求 → 7. 处理响应
        ↓
8. 过滤短剧内容 → 9. 去重 → 10. 排序 → 11. 分页
        ↓
12. 返回响应

        ↓ (如果无结果)
13. 使用通用关键词再次搜索 → 回到步骤 6

        ↓ (如果仍然无结果)
14. 返回空结果
```

## 11. 总结

短剧数据获取采用了多站点、多关键词的并行搜索策略，结合严格的数据过滤和高效的数据处理，确保能够获取到丰富、准确的短剧内容。整个流程设计考虑了性能优化和错误处理，能够应对各种边缘情况，为用户提供稳定的短剧数据服务。
